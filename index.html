silov0 API docs

An AI news feed delivered as structured JSON. These docs mirror a typical Docusaurus layout so you can browse concepts, endpoints, and a live data demo in one place.

Static HTML powered by posts/posts.json.

Swap in your own backend to ship it as a real API.

Overview

#overview

silov0 condenses AI release notes, product updates, and applied research takeaways into concise posts. Each entry has a title, summary, date, and optional link, making it easy to surface as a changelog or feed.

The static site doubles as an API façade: posts/posts.json is read client-side and rendered into cards and tables. If you replace the JSON source with a hosted endpoint, this layout becomes a production-ready docs surface.

Authentication

#authentication

The bundled demo runs without auth because it fetches a local JSON file. In a deployed service, requests would include an Authorization: Bearer <api_key> header. Keys gate read access and prevent unauthorized mutations.

Tip: Store API keys on the server when proxying requests. Client-side keys should be limited to public, cacheable reads.

Resources & endpoints

#endpoints

silov0 exposes two core resources: posts (the news items) and tags (topic labels). Use tags to cluster updates, or search across titles and summaries to surface relevant items.

GET /posts

Returns the full feed of posts with metadata and tag arrays.

  • ?tag=models filters to posts that include the tag.
  • ?q=latency searches titles and summaries.
  • ?limit=20 caps page size in a hosted deployment.

GET /posts/{id}

Retrieves a single post by identifier, useful for deep links.

GET /tags

Lists all known tags and counts so you can build filters.

Example request
curl "https://api.silov0.dev/posts?tag=models&q=latency" \
  -H "Authorization: Bearer sk_test_123"
Sample response
{
  "posts": [
    {
      "id": "launch-42",
      "title": "New model latency tuning",
      "summary": "Cut median latency by 18% with better batching",
      "date": "2024-04-18",
      "tags": ["models", "latency"],
      "url": "https://example.com/post"
    }
  ]
}
JavaScript fetch
async function loadPosts() {
  const res = await fetch('/posts?tag=evaluation');
  const { posts } = await res.json();
  return posts;
}

silov0 API docs

Structured AI news feed exposed as JSON-like resources.

Last updated May 2024

Overview

silov0 packages quick AI news drops, release notes, and applied takeaways into a structured stream. This static site stands in for the API surface that would normally power a client feed.

Everything is stored in posts/posts.json, rendered as cards and tables. Think of it as a developer-friendly mirror of a lightweight publishing backend.

Authentication

The live demo is unauthenticated because it reads local JSON. In a hosted setup, requests would include an Authorization: Bearer <api_key> header. Keys scope access to read-only news feeds and prevent mutation outside trusted publishers.

Rotate keys regularly and keep them server-side when proxying requests. Client-side usage should be limited to public, cacheable endpoints.

Resources & endpoints

silov0 models two primary resources: posts and tags. Posts contain titles, summaries, dates, and optional URLs. Tags cluster posts by topic, letting clients filter or prefetch by theme.

GET /posts

Returns the full feed of posts with metadata and tag arrays.

  • ?tag=models filters to posts that include the tag.
  • ?q=latency performs a case-insensitive search over title and summary.
  • ?limit=20 constrains page size in a hosted variant.

GET /posts/{id}

Retrieves a single post by identifier, useful for detail pages.

GET /tags

Lists known tags and counts, allowing clients to populate filters.

Filtering

Combine text search with tag filters to slice the feed. Queries like /posts?tag=models&q=latency surface latency-focused model updates. The UI below mirrors how a consumer app might expose these controls.
